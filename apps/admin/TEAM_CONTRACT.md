# Git Contract - Group 2

Everyone in the group approves this on 2026-02-19.

## Branch Naming

- main - always production-ready code
- dev - development branch
- feature/name-of-feature (e.g. feature/login-page)
- bugfix/name-of-bug (e.g. bugfix/fix-navbar-mobile)
- hotfix/quick-fix (only for urgent production issues)

## Commit Messages

- Start with imperative verb: "Add", "Fix", "Update", "Refactor", etc.
- Example: "Add user authentication with JWT"
- Link to issue when relevant: "Fix #13 - broken image upload"

## Pull Requests

- At least one other person must review and approve before merge
- Write a clear description: what, why, how you tested
- Use "Closes #X" or "Fixes #X" in the PR description to auto-close issues

## Other Rules

- Never push directly to main
- Pull from main frequently before creating a new branch
- Run tests/linter before committing/pushing (when we have them set up)
